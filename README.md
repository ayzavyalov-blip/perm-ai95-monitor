# АИ-95 Пермь: быстрый мониторинг

Статическая витрина для GitHub Pages + сбор данных через GitHub Actions.

## Что делает

1. По расписанию запускает `scripts/collect.py`.
2. Проверяет источники по АИ-95 в Перми:
   - gdebenz.ru
   - russiabase.ru/prices?city=154336
   - fuelprice.ru
   - benzin-price.ru
   - multigo.ru
   - benzup.ru
   - внешние источники, которые чаще требуют ручного просмотра: Т-Банк Топливо, Яндекс Заправки, Яндекс Карты, 2ГИС
3. Сохраняет результат в `data/latest.json`.
4. GitHub Pages показывает `index.html` и читает `data/latest.json`.

## Важное ограничение

GitHub Pages не выполняет серверный Python-код. Поэтому сбор данных нельзя делать из `index.html`.
Правильная схема: GitHub Actions собирает данные, GitHub Pages отображает готовый JSON.

## Быстрый запуск

1. Создать репозиторий на GitHub, например `perm-ai95-monitor`.
2. Загрузить содержимое этой папки в репозиторий.
3. В Settings -> Pages выбрать:
   - Source: Deploy from a branch
   - Branch: main
   - Folder: /root
4. В Actions вручную запустить workflow `Update AI-95 data`.
5. Открыть GitHub Pages URL.

## Режим обновления

По умолчанию workflow запускается каждый час на 17-й минуте:

```yaml
- cron: "17 * * * *"
```

17-я минута выбрана специально, чтобы не попадать в пик нагрузки в начале часа.

## Как трактовать данные

Статусы:
- `likely_available`: вероятно есть
- `maybe_available`: возможно есть
- `unknown`: не подтверждено
- `not_recommended`: не рекомендовать
- `source_unavailable`: источник не отдал пригодные данные

Цена не считается подтверждением наличия АИ-95.
Пробка у АЗС не считается самостоятельным подтверждением наличия АИ-95.

## Что доработать после первого запуска

1. Посмотреть `data/latest.json`.
2. Проверить, какие источники реально отдают HTML с таблицами.
3. Для каждого источника уточнить парсер в `scripts/collect.py`.
4. Если источник динамический, добавить ручной режим: пользователь присылает скрин или CSV.
5. Для маршрутов и пробок лучше использовать официальные API, если есть доступ.
